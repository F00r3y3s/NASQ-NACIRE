# Enhanced Workflow

## Stage 1: Trigger and Grounding
1. Gather the workflow source, artifact source, repo state, and any extra images or docs.
2. Inspect the actual repo and product assets before asking questions.
3. Separate:
   - visual source of truth
   - behavioral source of truth
   - workflow source of truth

## Stage 2: Research and Pattern Extraction
4. Review reference repos or docs only for process, planning, or implementation patterns.
5. Do not copy product content from reference materials.
6. Capture the useful patterns in a research note.

## Stage 3: Decision Capture
7. Lock high-impact decisions early.
8. Record every meaningful decision in a ledger so it is not reopened later.

## Stage 4: Formal Artifacts
9. Write the Epic Brief.
10. Write the Core Flows.
11. Run PRD Validation.
12. Write the Tech Plan.
13. Run Architecture Validation.
14. Write the Ticket Breakdown.
15. Run Cross-Artifact Validation.

## Stage 5: Execution Baseline
16. Materialize the planning docs into the repo.
17. Create research notes and handoff memory files.
18. Create the docs-grounding checklist.
19. Only then begin feature implementation.

## Stage 6: Controlled Execution
20. Implement ticket by ticket.
21. Re-check official docs for each ticket that touches framework or library behavior.
22. Run tests and security checks continuously.
23. Revise requirements only if execution reveals a real mismatch.
