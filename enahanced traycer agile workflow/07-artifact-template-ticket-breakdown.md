# Template: Ticket Breakdown

Use this structure for each ticket.

## Ticket Title
`Refs:` artifact references | `Depends:` dependencies  
`Included:` explicit scope  
`Excluded:` what is intentionally not part of this ticket  
`Acceptance:` the observable done condition

## Ticket Writing Rules
- Tickets should be implementation-ready, not vague stories.
- Each ticket should have clear boundaries.
- Dependencies should prevent out-of-order work.
- Acceptance criteria should be testable.
