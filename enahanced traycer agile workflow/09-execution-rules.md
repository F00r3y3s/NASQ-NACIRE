# Execution Rules

## Before Coding
- Materialize formal planning artifacts into the repo.
- Create research notes and handoff files.
- Confirm the visual source of truth and behavior source of truth.

## During Coding
- Implement ticket by ticket.
- Do not reopen settled product decisions unless execution reveals a real conflict.
- Keep a running execution status note.
- Prefer clean vertical slices over giant unbounded changes.

## Before Closing a Ticket
- Verify artifact fidelity.
- Verify tests and validation.
- Verify security and permission boundaries.
- Update handoff notes if the ticket changes important assumptions.
